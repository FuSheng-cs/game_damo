# 项目简介与 AI 原生创新说明

## 项目定位

《天台十句》不是一个“接了 LLM 的文字游戏”，也不是一个“包装成游戏的 AI 陪聊产品”。

它的目标是做一款 **AI 原生游戏**：游戏的核心规则、交互节奏、反馈形式、叙事判定，都是围绕大语言模型的能力与限制反向设计出来的。  
换句话说，这个项目不是先有一套传统游戏框架，再把 AI 塞进去；而是从一开始就把 LLM 视为游戏系统的一部分。

如果用一句话概括：

**《天台十句》是在探索：当 LLM 不再只是内容生成工具，而是直接参与玩法规则、情绪判定和叙事分歧时，游戏应该如何被重新设计。**

## 项目简介

玩家在深夜的天台遇见一个濒临崩溃的女孩“艾”，必须在有限的十句话之内与她交流，尝试阻止她坠落。

表面上，这看起来像一个高概念的情绪叙事作品；但项目真正的重点不只是题材，而是它如何把 AI 的特性转化为玩法结构：

- 玩家输入的不是选项，而是自然语言
- LLM 不是只负责回复一句台词，而是参与整个互动流程
- 游戏节奏不是预设好的，而是由模型响应速度、判定逻辑和上下文管理共同塑造
- 叙事分歧不是固定分支，而是由玩家表达与模型判断动态生成

因此，它更接近一种“围绕模型交互重新设计的游戏形态”，而不是传统意义上的文字冒险。

## 为什么做这个项目

我想解决的不是“怎么让 AI 角色更像人”，而是另一个更根本的问题：

**如果未来的游戏不是把 AI 当功能插件，而是把 AI 当底层交互材料，那么游戏规则本身应该怎么变？**

目前很多所谓 AI 游戏，本质上依然是：

- 固定剧情 + AI 文案润色
- 固定角色卡 + 无限陪聊
- 传统 UI + LLM 作为回复引擎

这类产品的问题是，AI 并没有真正进入游戏机制，只是承担了“让文本看起来更自然”的工作。

而我想做的是另一条路线：

- 正面承认 LLM 有成本问题
- 正面承认 LLM 有延迟问题
- 正面承认 LLM 更适合自然语言、模糊判断、动态生成和多模态扩展
- 再基于这些前提，反过来设计玩法、节奏和交互

所以《天台十句》的出发点，不是“做一个能和 AI 聊天的游戏”，而是“做一个只有在 AI 存在时才成立的游戏”。

## 这款作品的核心创新性

## 1. 把 Token 成本反向设计成玩法规则

“十句话内拯救一个女孩”首先是叙事设定，但它更深层的作用，是一个非常明确的 **AI 原生成本控制机制**。

LLM 游戏天然面临一个问题：

- 对话越自由，上下文越长
- 上下文越长，Token 成本越高
- 成本越高，延迟越明显，也越难持续在线运行

传统做法通常是：

- 截断上下文
- 强行缩短回复
- 或者直接回避真实自由对话

而《天台十句》的做法是，把这个技术问题直接上升为玩法设计：

- 玩家初始只有 10 次发言机会
- 每一句都变得昂贵且有意义
- 只有真正打动角色，才能获得额外回合

这样一来，“有限轮次”不再只是限制，而成为叙事压力、资源管理和成本控制三者统一的核心机制。

这就是 AI 原生设计最关键的一点：

**不是隐藏模型限制，而是把模型限制转化成游戏规则。**

## 2. 把 LLM 延迟反向设计成情绪等待

LLM 的另一个天然问题是响应延迟。

在普通产品里，延迟意味着卡顿；但在游戏里，如果处理得好，它可以变成体验的一部分。

因此，《天台十句》在玩家发送消息后，不是生硬地显示“Loading...”，而是用以下方式去消化延迟：

- 使用等待文案，如“她吐了一口烟圈……”
- 配合角色状态与场景氛围，让等待看起来像角色正在思考、抽烟、沉默
- 后续可以继续强化为抽烟 CG、动作过渡和环境镜头

这背后的设计逻辑不是 UI 修饰，而是：

**把模型计算时间包装成角色存在时间。**

也就是说，等待不再是系统空白，而是叙事中的停顿、观察和呼吸。  
这种处理方式能够把技术延迟转化为情绪延迟，让玩家觉得“她在想”，而不是“服务器在算”。

## 3. 打字机效果不是装饰，而是为流式输出准备的表现层

很多文字游戏使用打字机效果，只是为了复古或氛围。

但在这个项目里，打字机效果更重要的价值在于：它天然适配 LLM 的 **流式输出** 形态。

也就是说，这个效果不是为了“好看”，而是为了和 AI 的输出方式建立一致性：

- LLM 可以逐步吐字
- UI 可以逐字呈现
- 玩家会感知到角色像是在实时组织语言，而不是一次性吐出整段文本

这意味着：

- 表现层和模型层的节奏可以统一
- 流式输出未来可以自然接入，不需要推翻现有交互逻辑
- 玩家会更强地感受到“她正在说话”，而不是“页面刷出一段结果”

因此，打字机效果在这里本质上是一个 **为 AI 流式交互预留的界面协议**。

## 4. LLM 参与的不是单点回复，而是多个判定阶段

这个项目最重要的一点，是 LLM 参与的不只是“回答”，而是多个游戏阶段的判断。

目前它已经介入了至少三个关键节点：

- **提示生成**：当玩家卡住时，模型不是给固定攻略，而是根据上下文给出方向性引导
- **角色回复**：角色不是固定台词，而是根据玩家表达实时回应
- **结局判定**：模型不仅生成语句，还通过标签参与好感增长与结局触发

这和普通 AI 对话产品有根本区别。

普通产品中，LLM 通常只负责“说什么”；  
而在《天台十句》中，LLM 还在参与“现在发生了什么”和“这一局最终算什么结果”。

也就是说，LLM 在这里已经不只是叙事生成器，而是一个 **判别器 + 生成器的混合游戏系统**。

## 5. 它追求的是真实感，不是固定分支感

传统分支叙事游戏的问题在于：

- 结构清晰
- 但很容易被玩家识别为“选对答案”

而普通 AI 陪聊的问题在于：

- 自由度很高
- 但没有清晰的代价、反馈和关卡感

《天台十句》的设计目标，是在两者之间找到第三种形态：

- 玩家用自然语言输入
- 角色反应不再是固定分支
- 但系统仍然有明确的回合、反馈和结局

这样会让体验更加接近真实沟通：

- 你不能只靠背答案通关
- 你必须感知对方情绪和语气变化
- 你需要承担自己说错话的后果

这种“真实感”并不是靠文案堆出来的，而是靠 **LLM 参与判定后的不确定性** 建立起来的。

## 和普通 AI 对话游戏的区别

很多同类产品本质上还是“AI 聊天界面 + 角色设定”。

《天台十句》和它们的核心差异在于：

### 1. 它不是无限对话，而是有限回合

无限对话更像陪伴产品；有限回合才会形成真正的关卡、压力和失败条件。

### 2. 它不是把 AI 当内容补全，而是把 AI 当规则参与者

这里的 AI 不只是生成一句回复，而是在参与：

- 交互节奏
- 情绪反馈
- 资源增减
- 结局判定

### 3. 它不是把技术问题藏起来，而是把技术问题设计进玩法

- Token 成本对应有限回合
- 响应延迟对应情绪等待
- 流式输出对应打字机表现

这三点，都是非常明确的 AI 原生设计，而不是传统游戏里常见的后贴式接入。

## 它解决了什么具体问题

从研究和产品角度看，这个项目解决的是三个具体问题：

### 1. 如何让 LLM 游戏在成本上成立

通过有限回合和增量奖励，把高自由度对话控制在一个可运营、可设计的范围内。

### 2. 如何让 LLM 延迟不破坏沉浸

通过角色动作、等待文本和后续可扩展的 CG 演出，把“模型在算”转化为“角色在想”。

### 3. 如何让 AI 真正进入游戏机制

通过提示、回答、结局判定等多个阶段的模型参与，让 AI 不再只是会写字，而是成为规则系统的一部分。

## 为什么说这是 AI 原生游戏，而不是 AI 对话游戏

AI 对话游戏的重点通常是：

- 角色像不像真人
- 回复够不够自然
- 人设是否稳定

而 AI 原生游戏更关心的是：

- 规则是不是围绕模型能力构建的
- 节奏是不是围绕模型延迟构建的
- 反馈是不是围绕模型判定构建的
- 扩展是不是围绕模型多模态特性构建的

《天台十句》的重点明显属于后者。

它不是在证明“AI 可以陪你聊天”，而是在证明：

**AI 可以成为游戏规则、游戏节奏和游戏现实感的一部分。**

## 未来方向

这个项目后续最重要的扩展，不是简单增加更多台词，而是继续沿着 AI 原生方向深化。

### 1. 多模态扩展：微信电话功能

后续计划加入类似微信电话的能力，让玩家不只是文本沟通，而是进入更强的临场交互形式。  
这会让“她像真的存在”这件事更进一步。

### 2. 多模态扩展：图片上传功能

玩家未来可以上传图片，角色结合多模态模型理解图像内容并做出反应。  
这意味着互动不再局限于打字，而是进入更接近真实社交关系的沟通方式。

### 3. 完全由 LLM 生成的结局

当前项目已有明确结局判定机制，但后续希望进一步把结局开放给模型生成。  
目标不是只有几个预定义结局名称，而是让不同玩家因为不同表达方式，真正打出不同的专属结局。

这会把项目从“多结局游戏”进一步推进到“生成式结局游戏”：

- 每个人的结局可以不同
- 每段关系可以不同
- 每次游玩都可能形成新的情绪收束方式

## 项目的评审价值

如果站在评委视角，这个项目的价值不只在于题材表达，而在于它提出了一套比较清晰的 AI 原生游戏设计思路：

### 1. 它把模型限制变成了玩法资产

不是回避 Token 和延迟问题，而是把它们转化为规则与演出的一部分。

### 2. 它把模型从文案工具升级为系统参与者

LLM 不只负责写对白，而是参与提示、回应、判定、结局这些关键阶段。

### 3. 它为多模态交互留出了自然延展空间

从文本到电话、图片、关系持续性，这条扩展路径是连续的，而不是临时拼接的。

### 4. 它在探索一种新的游戏形态

不是传统意义上的 AVG，也不是普通酒馆产品，而是一种以模型能力为底层前提重新组织玩法的游戏原型。

## 结论

《天台十句》真正想证明的不是“AI 可以写出更自然的对白”，而是：

**当模型能力、模型延迟、模型成本、模型判定和模型多模态能力都被当成原生条件时，游戏本身会长成另一种样子。**

所以它的创新点不在“用了 AI”，而在于：

- 用十句话机制处理 Token 成本
- 用等待演出处理 LLM 延迟
- 用打字机效果适配流式输出
- 用模型参与提示、回复和结局判定
- 用未来多模态能力继续扩展“像真实存在的人”这一交互目标

这也是这个项目最值得被看见的地方：

**它不是在给游戏接 AI，而是在尝试为 AI 重新发明游戏。**
